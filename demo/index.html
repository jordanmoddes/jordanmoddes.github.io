The only thing you need to know

One formula. Every page.

GitHub Pages has a magic trick. The name of the folder you create becomes the URL path. That's it. That's the whole secret.

📐 The Formula
rooms / index.html → yoursite.com/rooms

Folder name = URL path  ·  index.html inside it = the actual page.
Make a folder. Drop an index.html inside it. GitHub does the rest.

📁 In your repo
contact/
index.html
→
🌐 On the web
yoursite.com/contact
💡
Why index.html specifically?

Web servers are trained to look for a file called index.html when someone visits a folder URL. It's been a universal web rule since the 90s — GitHub follows it too.

Step by step

How to add a new page

Follow these 4 steps every single time you want a new page on your site.

1
Go to your repository on GitHub

Head to github.com, sign in, and click on your website's repository. It'll look something like yourname.github.io.

🐙
github.com / yourname / yourname.github.io Your main website repository
📦 yourname.github.io Public
📁 about add about page
📄 index.html initial commit
📄 style.css initial commit
2
Click "Add file" → "Create new file"

At the top right of your file list, click the Add file button and choose Create new file. A text editor will open with an empty filename box at the top.

👆
The filename box is where the magic happens.

What you type there determines both the folder name AND the URL. Keep reading — it's super simple.

3
Type your folder name, a slash, then "index.html"

In the filename box, type the page name you want, then a forward slash /, then index.html. The moment you type the slash, GitHub automatically turns it into a folder. ✨

Name your file
yourname.github.io /
rooms/index.html
📦 yourname.github.io
📁 about existing
📁 rooms ← just created!
📄 index.html home page
⚡
Type the slash to create the folder automatically.

You don't create folders separately on GitHub. Just type foldername/index.html and GitHub handles it. The folder appears the moment you type /.

🚫
Keep folder names simple.

Lowercase letters, hyphens instead of spaces. Examples: my-rooms, contact, book-now. No spaces, no capitals.

4
Paste in your HTML and commit

In the big editing area below the filename, paste your page's HTML code. Then scroll down, click "Commit new file", and you're done. GitHub will publish it automatically.

rooms/index.html · starter template 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Rooms — My Airbnb</title>
</head>
<body>

  <h1>Our Rooms 🛏️</h1>
  <p>Describe your space here!</p>

  <!-- Put your content here -->

</body>
</html>
⏱️
Wait ~60 seconds after committing.

GitHub needs a moment to rebuild your site. Give it a minute, then visit the URL. If it's not up yet, wait another 30 seconds and refresh.

★
First time? Enable GitHub Pages (one time only)

You only do this once, ever. Go to your repo → Settings → Pages. Under "Source," choose your main branch. Hit Save. Done forever.

🔗
Link between pages

On any page you can add a link with <a href="/rooms">See Rooms</a>. Use the URL path (starting with /) and GitHub handles the rest.

🎨
Sharing one stylesheet across all pages

Put style.css in your root folder. Reference it from any subfolder page with <link rel="stylesheet" href="/style.css">. The leading / means "start from the root."
